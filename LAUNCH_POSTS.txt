Title: I built an open-source AI gateway in Go — routes, rate-limits, and secures LLM traffic across providers

Body:

Hey r/golang,

I just released AegisFlow, an open-source AI gateway written in Go. It sits between your applications and LLM providers (OpenAI, Anthropic, Ollama, etc.) and handles routing, rate limiting, security policies, and observability.

**What it does:**
- OpenAI-compatible API — point any OpenAI SDK at it by changing `base_url`
- Multi-provider routing with automatic fallback and circuit breaker
- Policy engine that blocks prompt injection and detects PII before it reaches providers
- Per-tenant rate limiting (sliding window, in-memory or Redis-backed)
- Usage tracking with token counts and cost estimation
- Prometheus metrics + OpenTelemetry tracing
- SSE streaming support

**Why Go:**
This is infrastructure that sits in the hot path of every AI request. Go gives me a single binary (~15MB), handles concurrent connections efficiently, and is what the cloud-native ecosystem expects for this kind of tool. Same reason Envoy alternatives, Traefik, and Kubernetes controllers are written in Go.

**Tech details:**
- chi router for HTTP
- Clean internal package boundaries (provider interface, middleware chain, policy engine)
- 40 unit tests, all passing with `-race`
- Works with local Ollama models — no API keys needed
- Docker + Docker Compose included

GitHub: https://github.com/saivedant169/AegisFlow

Would love feedback on the architecture and code quality. Issues are open for contributions — several `good first issue` labels for anyone who wants to add a provider adapter.

Title: AegisFlow — self-hosted AI gateway that routes, rate-limits, and secures your LLM API calls

Body:

I built an open-source AI gateway you can self-host. It sits between your apps and AI providers, giving you control over routing, security, costs, and monitoring.

**The problem it solves:**
If you're running multiple AI-powered tools (chatbots, copilots, automation), each one talks directly to OpenAI or whatever provider. No central visibility, no rate limits, no security scanning, no failover. If OpenAI goes down, everything breaks.

**What AegisFlow does:**
- All AI traffic flows through one gateway
- Route to OpenAI, Anthropic, Ollama (local models), or any OpenAI-compatible API
- Automatic failover — if primary provider fails, routes to backup
- Rate limit per user/team
- Block prompt injection attempts before they reach the AI
- Detect PII (emails, SSNs, credit cards) in prompts
- Track usage and costs per tenant
- Prometheus metrics for monitoring

**How to run it:**
```
git clone https://github.com/saivedant169/AegisFlow.git
cd AegisFlow
docker compose -f deployments/docker-compose.yaml up
```

Or without Docker:
```
make run
```

Works with Ollama out of the box — no API keys needed.

Single Go binary, ~15MB, runs on anything.

GitHub: https://github.com/saivedant169/AegisFlow

Title: Built an AI gateway that works with Ollama — adds rate limiting, prompt security, and usage tracking to your local models

Body:

I've been running Ollama locally and wanted the same kind of controls that cloud AI APIs have — rate limiting, prompt injection blocking, usage tracking, multi-model routing. So I built AegisFlow.

It's an AI gateway (written in Go) that sits in front of Ollama (or any provider) and adds:

- **OpenAI-compatible API** — any app that works with OpenAI works with AegisFlow + Ollama, just change the base URL
- **Prompt injection blocking** — catches "ignore previous instructions" before it reaches your model
- **PII detection** — warns if someone sends emails or SSNs to the model
- **Usage tracking** — see how many tokens each user consumed
- **Multi-model routing** — route different model names to different Ollama models or cloud providers
- **Failover** — if Ollama is down, fall back to a cloud provider (or mock)

Tested with qwen2.5:0.5b on M1 Mac. Works great.

No API keys needed. Fully local. Apache 2.0.

```
# Start AegisFlow
make run

# Chat through the gateway (uses Ollama)
curl -X POST http://localhost:8080/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "X-API-Key: aegis-test-default-001" \
  -d '{"model":"qwen2.5:0.5b","messages":[{"role":"user","content":"Hello!"}]}'
```

GitHub: https://github.com/saivedant169/AegisFlow
